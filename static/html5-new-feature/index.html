网站访问性能提升最佳实践

核心：最小化HTTP请求 前端占终端用户响应时间达80%，而其中大部分的时间用于下载网页内容，例如图片，样式表，脚本和flash等。降低组件数量可以减少为渲染网页而发出的HTTP请求，也就是网页加速的关键。 其中一条可行的方法是简化网页设计，减少组件数量。但问题是是否可以构架具有丰富内容同时具有快速响应特性的网页呢？以下几个技巧就能帮我们实现快速、丰富内容的网页： 合并文件(combined files) 当网页的行为js文件与文档结构分离时，使用script标签引入外部js文件。 包含脚本的最佳方式是引入外部文件，而且浏览器也能对站点中的多个页面重用缓存过的相同脚本，但涉及引入多个文件时： 推荐的做法是将上述脚本文件合并到一个脚本文件中。这样，就可以减少加载页面时发送的请求数量。而减少请求数量往往是性能优化时优先考虑的问题。 通过合并所有脚本文件至一个文件减少HTTP请求数量，样式表也是同理。然而合并不同页面包含不同的脚本和样式表会具有相当大的挑战性，但采用文件合并可降低响应时间。 CSS Sprites CSS
