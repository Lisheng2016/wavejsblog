Javascript创建数组:"new Array()"和"[]"的区别

by admin

2016-11-16
用过:

var myArray = new Array();

也用过:

var myArray = [];

那么：区别在哪里？ 速度有区别吗？

1.具体有什么区别？

就声明一个空白数组(即上面的声明方式)而言这两种没有区别。但是在实际应用中还是有些区别，正是因为这些区别，我认为这两种方式中应该偏向用：

var myArray = [];

new Array()是比较语义化的创建方式，new Array(length)的参数length声明数组长度，然后生成该长度的空数组。

myArray = new Array(5);
alert(myArray.length); // 5

而下面的例子中,两者略有不同:

var a = [], // these are the same
b = new Array(), // a and b are arrays with length 0

c = [‘foo’, ‘bar’], // these are the same
d = new Array(‘foo’, ‘bar’), // c and d are arrays with 2 strings

// these are different:
e = [3] // e.length == 1, e[0] == 3
f = new Array(3), // f.length == 3, f[0] == undefined
;

而实际上，两种创建数组的方式在编程中又有微妙但是重要的区别：

var a = [];

直接创建数组，调用解释器创建，没有额外的部分参与创建过程。

var a = new Array();

调用Array()创建数组，看起来没什么区别，但是万一程序中Array()被重新定义，那就不一样：

function Array() {
this.is = ‘SPARTA’;
}

var a = new Array();
var b = [];

alert(a.is); // => ‘new Array’
alert(b.is); // => undefined
a.push(‘Woa’); // => TypeError: a.push is not a function
b.push(‘Woa’); // => 1 (OK)

第一个输出’new Array’，第二个不会；当利用push时，也会发现两者的不同。

这个例子只是说明Array()被重新定义带来的问题，其实并不常见。

More:
这篇文章链接中提到：

You never need to use new Object() in JavaScript. Use the object literal {} instead. Similarly, don’t use new Array(), use the array literal [] instead. Arrays in JavaScript work nothing like the arrays in Java, and use of the Java-like syntax will confuse you.

Do not use new Number, new String, or new Boolean. These forms produce unnecessary object wrappers. Just use simple literals instead.
