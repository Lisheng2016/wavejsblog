:nth-child 与 :nth-of-type的区别

Little

Piggy

假设我想将第二个Paragraph元素设置属性：

p:nth-child(2) { color: red; }

p:nth-of-type(2) { color: red; }

显然有区别：
第一个选择的含义是：
选择父元素的第二个子元素的p元素，条件需要同时满足！

第二个选择的含义是:
选择第二个p元素

那么原先的情况有发生变化：

p:nth-child(2) { color: red; } /* 不正确 */

p:nth-of-type(2) { color: red; } /* 仍正确 */

:nth-child选择器选择了第一个p元素，但是需要改的是第二个，由于确实是父元素的第二个p子元素，实际上并没有错，如果在h1元素下加入一个h2元素，那么不会选择任何元素，而:nth-of-type选择的元素依然是第二个p元素，那么还是正确的

因此,如果只是简单应用，那么:nth-of-type更加精准，:nth-child要好用，涉及单双数或者其他情况, 那么:nth-child要有用的多

除了:nth-of-style :nth-child之外：
还有

:root – 选择整个文档的根元素，基本上是HTML元素，XML环境下会不一样

:first-child – 选择父元素的第一个子元素

:last-child – 选择父元素的最后一个元素

:nth-child() – 可以实施具体的筛选，比如:2n 4n-1等

:nth-of-type() – 父元素下具体筛选某种元素，还可以加入odd筛选奇数项

:first-of-type – 选择父元素下第一个是该类型的元素

:last-of-type – 选择父元素下最后一个是该类型的元素

:nth-last-of-type() – 类似:nth-of-type,排序从底部向上

:nth-last-child() – 类似:nth-child,排序从底部向上

:only-of-type – 当父元素下该类型元素只有一个才会被选择
