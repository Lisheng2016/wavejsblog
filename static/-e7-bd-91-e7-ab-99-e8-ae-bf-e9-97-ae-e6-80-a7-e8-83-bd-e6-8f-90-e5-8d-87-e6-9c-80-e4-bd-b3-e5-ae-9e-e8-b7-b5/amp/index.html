网站访问性能提升最佳实践

by admin

2016-11-07

核心：最小化HTTP请求

前端占终端用户响应时间达80%，而其中大部分的时间用于下载网页内容，例如图片，样式表，脚本和flash等。降低组件数量可以减少为渲染网页而发出的HTTP请求，也就是网页加速的关键。

其中一条可行的方法是简化网页设计，减少组件数量。但问题是是否可以构架具有丰富内容同时具有快速响应特性的网页呢？以下几个技巧就能帮我们实现快速、丰富内容的网页：

合并文件(combined files)

当网页的行为js文件与文档结构分离时，使用script标签引入外部js文件。

包含脚本的最佳方式是引入外部文件，而且浏览器也能对站点中的多个页面重用缓存过的相同脚本，但涉及引入多个文件时：

推荐的做法是将上述脚本文件合并到一个脚本文件中。这样，就可以减少加载页面时发送的请求数量。而减少请求数量往往是性能优化时优先考虑的问题。
通过合并所有脚本文件至一个文件减少HTTP请求数量，样式表也是同理。然而合并不同页面包含不同的脚本和样式表会具有相当大的挑战性，但采用文件合并可降低响应时间。

CSS Sprites

CSS Sprites是一种处理网页图片的方法，通过将该页面所有涉及到的图片合并到一张图片中，再利用CSS的“background-image”，“background- repeat”，“background-position”的组合进行背景定位，background-position可以用数字精确的定位出背景图片的位置。

### CSS Sprite的例子

.bg_sprite{background-image:url(/整图地址); background-repeat:no-repeat} //引用该类 #ico1 {width:容器宽度;height:容器高度;background-position:X坐标 Y坐标} #ico2 {width:容器宽度;height:容器高度;background-position:X坐标 Y坐标} #ico3 {width:容器宽度;height:容器高度;background-position:X坐标 Y坐标} .nav {width:容器宽度;height:容器高度;background-position:X坐标 Y坐标} //元素逐一定义背景坐标

## Image maps

(online creation site)
Image map将多张图片整合到一张图片。整体大小几乎没有变化，但减少HTTP请求，加速网页访问速度。只有页面内图片连续比如导航条时，才可以使用。定义图片坐标很繁琐而且容易出错。image maps制作的导航条也是不可访问的，所以现在不推荐用Image maps

Inline Images

利用data:URL scheme将图片数据嵌入页面。嵌入图片会增加文档的大小，所以将inline image嵌入(已缓存)样式表可以减少请求，同时避免增加页面文件的大小。但现在inline image的主流浏览器支持还有所欠缺。

总结

我们必须时刻牢记，加速的关键是减少HTTP请求，同时这也极大的提升了第一次访问该页面的用户的用户体验，以前看过一篇Tenni Theurer的博客文章 Browser Cache Usage – Exposed!，40-60%的用户访问站点时是没有缓存的，因此加强页面首次加载的能力是提升用户体验的关键。
