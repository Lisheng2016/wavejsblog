对Sass的一些理解

1.Mixin和Placeholder区别是什么?使用时的最佳实践是怎么样的?

刚开始学习SASS预处理器时,我对mixin和placeholder的区别不太了解,其实经过比较,两者还是很容易区分的,也可以说两者应用的条件其实区分明确,但又可以交叉,看需求和方便程度自由选择即可.

1.1 Mixin it up

一个mixin可以包含多条可能需要反复利用的CSS规则,类似函数只不过是现在调用它输出的是CSS规则,以下是Sass参考:

Mixins allow you to define styles that can be re-used throughout the stylesheet without needing to resort to non-semantic classes like .float-left. Mixins can also contain full CSS rules, and anything else allowed elsewhere in a Sass document. They can even take arguments which allows you to produce a wide variety of styles with very few mixins.

下面举几个例子关于参考中描述的Sass mixin的实际作用:

@mixin center() { display: block; margin-left: auto; margin-right: auto; } .container { @include center(); /* Other styles here... / } / Other styles... */ .image-cover { @include center; }

那么从例子中也能看得出,不一定要向mixin传递参数,定义的时候也可以不定义参数,如此步骤只需要将重复的CSS放在一个mixin里重复利用,减少了工作量.

再比如,类似定义width,height的常见CSS也可以利用mixin,不过这个是要有参数的:

@mixin size($width, $height: $width) { width: $width; height: $height; }

使用时只要调用@include size(width,height);(默认没有height参数会使用width的值)

.icon { @include size(32px); } .cover { @include size(100%, 10em); }

同样的,定位也可以用mixin方法完成,方便快捷:

@mixin position($position,$top:0px,$right:0px,$bottom:$top,$left:$right){ position:$position; top:$top; right:$right; bottom:$bottom; left:$left; }

1.2 Placeholder

Placeholder的类就是给存在样式交叉的元素共同定义用的,以%开头,本身不参加CSS编译,也不会在编译结果中出现,以下举例说明一下:

%center { display: block; margin-left: auto; margin-right: auto; }

应用时在需要其包含CSS规则的类中用@extend”引入”:

.container1 { @extend %center; } .container2 { @extend %center; }

那么实际上.container1,.container2的编译内容不会变化,只是编译后%center处被替代为.container1,.container2 {…},相当于取代%center的位置,替其”追加”属性.

而@extend还能够”引入”其他整个CSS类:(向.table-zebra”追加”.table的规则)

.table-zebra { @extend .table; tr:nth-of-type(even) { background: rgba(0,0,0,.5); } }

1.3 如何选择？

以上两者都可以减少不必要的重复劳动,那么区分使用情况的根据来源于它们的特性:
placeholder里不能用参数,无法具体修改其某一项规则,只能追加预定义规则
mixin如果不使用参数,纯粹使用@include引入会造成大量代码冗余:

没有参数的mixin:

@mixin center { display: block; margin-left: auto; margin-right: auto; } .container { @include center; } .image-cover { @include center; }

其编译结果为:

.container { display: block; margin-left: auto; margin-right: auto; } .image-cover { display: block; margin-left: auto; margin-right: auto; }

同样情况使用placeholder:

%center { display: block; margin-left: auto; margin-right: auto; } .container { @extend %center; } .image-cover { @extend %center; }

其编译结果为:

.container, .image-cover { display: block; margin-left: auto; margin-right: auto; }

可见,placeholder不会造成代码的重复(当前情况下).

当然,根据情况不同,可以同时用placeholder和mixin,比如我想要a,b两个类,包含不一样的颜色和字体大小,但具有相同的居中和块元素属性:

%center { margin-left: auto; margin-right: auto; display: block; } @mixin skin($color, $size) { @extend %center; background: $color; height: $size; } a { @include skin(pink, 10em) ；@extend %center;} b { @include skin(blue, 90px) ；@extend %center;}

编译结果为:

a, b { margin-left: auto; margin-right: auto; display: block; } a { background: pink; height: 10em; } b { background: blue; height: 90px; }

1.4 总结

需要具体化参数请使用mixin,宽泛的重复利用placeholder足矣

2.Sass的一些计算规则

2.1 加法

在 CSS 中能做运算的,到目前为止仅有 calc() 函数可行.但在 Sass 中,运算只是其基本特性之一
sass做加法运算是可以不考虑参数带单位,但需要单位同一
加号可以不需要空格隔开

$sidebar-width: 220px; $content-width: 720px; $gap-width: 20px; .container { width: $sidebar-width+$content-width + $gap-width; margin: 0 auto; } //输出结果为: .container { width: 960px; margin: 0 auto; }

2.2 减法 – 需要注意空格

$full-width: 960px; $sidebar-width: 200px; .content { width: $full-width - $sidebar-width; } //输出结果: .content { width: 760px; }

2.3 乘法

能够支持多种单位(比如 em ,px , %)；

如果进行乘法运算时,两个值单位相同时,只需要为一个数值提供单位即可(多个乘数中只需要一个乘数提供单位,否则报错).

.box { width: 10px * 2; } //输出结果: .box { width: 20px; }

2.4 除法

众所周知“/”符号在 CSS 中已做为一种符号使用.因此在 Sass 中做除法运算时,直接使用“/”符号做为除号时,将不会生效,编译时既得不到我们需要的效果,也不会报错.
需要给运算的外面添加一个小括号()才能执行除法运算

.box {
width: (100px / 2);
}

总结: ”/”符号被当作除法运算符时有以下几种情况:

- 如果数值或它的任意部分是存储在一个变量中或是函数的返回值

  • 如果数值被圆括号包围

  • 如果数值是另一个数学表达式的一部分

p { font: 10px/8px; // 纯 CSS,不是除法运算 $width: 1000px; width: $width/2; // 使用了变量,是除法运算 width: round(1.5)/2; // 使用了函数,是除法运算 height: (500px/2); // 使用了圆括号,是除法运算 margin-left: 5px + 8px/2px; // 使用了加(+)号,是除法运算 } //输出结果: p { font: 10px/8px; width: 500px; width: 1; height: 250px; margin-left: 9px; }

Sass 的除法运算还有一个情况.先回忆一下,在乘法运算时,如果两个值带有相同单位时,做乘法运算时,出来的结果并不是我们需要的结果.但在除法运算时,如果两个值带有相同的单位值时,除法运算之后会得到一个不带单位的数值

.box { width: (1000px / 100px); } //输出结果: .box { width: 10; }

2.5 颜色运算 – 分段运算

所有算数运算都支持颜色值,并且是分段运算的.也就是说,红、绿和蓝各颜色分段单独进行运算

p { color: #010203 + #040506; } 计算公式为 01 + 04 = 05、02 + 05 = 07 和 03 + 06 = 09,然后合并 p { background-color: #012345 + #543210; color:#123456 * 2; } //输出结果: p { background-color: #555555; color: #2468ac; }
