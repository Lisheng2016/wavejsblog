Centos7优化安装Shadowsocks

本站基于Vesta面板安装，Centos7默认开启firewalld 服务,Vesta面板默认安装iptables防火墙，需要我们预先添加规则，shadowsocks才可以正常工作！

1.关闭Centos7防火墙> $ systemctl stop firewalld

$ systemctl disable firewalld

如果提示firewalld不存在，无需理会，进入下一步即可

2.安装shadowsocks

$ yum install -y python-setuptools m2crypto && easy_install pip
$ pip install shadowsocks

yum命令添加 －y参数不再询问是否允许软件安装的请求

3.创建并修改shadowsocks配置文件

$ touch /etc/shadowsocks.json //创建shadowsocks.json文件
$ vi /etc/shadowsocks.json //修改shadowsocks配置文件

{ "server":"Your server IP", "port_password": { "port1": "password1", //port1是shadowsocks使用的端口1,可自行修改 "port2": "password2" //port2是shadowsocks使用的端口2,可自行修改 }, //端口可以继续添加，不必止于端口2 "timeout":400, //连接超时时间 "method":"aes-256-cfb", //加密方式 "fast_open": false //加速TCP连接，默认为False,稍后优化配置将其置为True }

4.启动ShadowSocks并测试是否可用

$ ssserver -c /etc/shadowsocks.json -d start
//以shadowsocks.json中的配置启动shadowsocks服务端
$ ssserver -c /etc/shadowsocks.json -d stop
//停止shadowsocks服务端

5.安装supervisor

$ easy_install supervisor
$ echo_supervisord_conf > /etc/supervisord.conf
$ vi /usr/lib/systemd/system/supervisord.service

需要添加至supervisord.service文件的原始文本

supervisord service for systemd (CentOS 7.0+) # by ET-CS (https://github.com/ET-CS) [Unit] Description=Supervisor daemon [Service] Type=forking ExecStart=/usr/bin/supervisord ExecStop=/usr/bin/supervisorctl $OPTIONS shutdown ExecReload=/usr/bin/supervisorctl $OPTIONS reload KillMode=process Restart=on-failure RestartSec=42s [Install] WantedBy=multi-user.target

$ vi /etc/supervisord.conf

[include] files = /etc/supervisord.d/*.conf

$ systemctl enable supervisord.service

6. 用supervisor管理shadowsocks

$ mkdir /etc/supervisord.d
$ vi /etc/supervisord.d/shadowsocks.conf

[program:shadowsocks] command=ssserver -c /etc/shadowsocks.json autorestart=true user=root

$ supervisord //启动supervisord服务端
$ supervisorctl reload //重载supervisorctl
$ supervisorctl start shadowsocks //运行SS
$ supervisorctl stop shadowsocks //停止SS
$ supervisorctl restart shadowsocks //重启SS

7.优化内核参数

$ ulimit -n 51200
$ vi /etc/security/limits.conf

root soft nofile 51200 root hard nofile 51200

$ vi /etc/sysctl.conf

max open files fs.file-max = 51200 # max read buffer net.core.rmem_max = 67108864 # max write buffer net.core.wmem_max = 67108864 # default read buffer net.core.rmem_default = 65536 # default write buffer net.core.wmem_default = 65536 # max processor input queue net.core.netdev_max_backlog = 4096 # max backlog net.core.somaxconn = 4096 # resist SYN flood attacks net.ipv4.tcp_syncookies = 1 # reuse timewait sockets when safe net.ipv4.tcp_tw_reuse = 1 # turn off fast timewait sockets recycling net.ipv4.tcp_tw_recycle = 0 # short FIN timeout net.ipv4.tcp_fin_timeout = 30 # short keepalive time net.ipv4.tcp_keepalive_time = 1200 # outbound port range net.ipv4.ip_local_port_range = 10000 65000 # max SYN backlog net.ipv4.tcp_max_syn_backlog = 4096 # max timewait sockets held by system simultaneously net.ipv4.tcp_max_tw_buckets = 5000 # TCP receive buffer net.ipv4.tcp_rmem = 4096 87380 67108864 # TCP write buffer net.ipv4.tcp_wmem = 4096 65536 67108864 # turn on path MTU discovery net.ipv4.tcp_mtu_probing = 1 # for high-latency network #net.ipv4.tcp_congestion_control = hybla # for low-latency network net.ipv4.tcp_congestion_control = htcp # turn on TCP Fast Open on both client and server side net.ipv4.tcp_fastopen = 3

$ modprobe tcp_hybla (&& modprobe tcp_htcp)
$ sysctl -p

根据自己的网络情况选用hybla或者htcp，高延迟网络(如美国，欧洲)使用hybla，低延迟网络(如日本，香港)使用htcp

8.加密算法更换为chacha20

$ yum install -y wget
$ yum groupinstall -y “Development Tools”
$ wget https://download.libsodium.org/libsodium/releases/LATEST.tar.gz
$ tar zxf LATEST.tar.gz
$ cd libsodium*
$ ./configure
$ make && make install

$ echo /usr/local/lib > /etc/ld.so.conf.d/usr_local_lib.conf //修复关联
$ ldconfig //修复关联

9.更改shadowsocks配置文件

$ vi /etc/shadowsocks.json

{ "server":"Your server IP", //申请的服务器ip地址 "port_password": { "port1": "password1", //port1是shadowsocks使用的端口1,可自行修改 "port2": "password2" //port2是shadowsocks使用的端口2,可自行修改 }, //端口可以继续添加，不必止于端口2 "timeout":400, //连接超时时间 "method":"chacha20", //加密方式->chacha20 "fast_open":true //加速TCP连接，默认为False,优化配置后置为True }

$ supervisorctl restart shadowsocks

10.安装锐速

$ wget -N –no-check-certificate https://raw.githubusercontent.com/91yun/serverspeeder/master/serverspeeder-all.sh && bash serverspeeder-all.sh
//安装锐速时会提示安装最接近的Linux内核版本,同意即可

11.防火墙放行目标端口

$ iptables -A INPUT -p tcp –dport port1 -j ACCEPT //port1对应shadowsocks配置的端口1
$ iptables -A INPUT -p tcp –dport port2 -j ACCEPT //port2对应shadowsocks配置的端口2
$ service iptables restart //重启iptables服务
$ iptables -L -n //查看规则是否成功添加

若开放的端口大于2个可以按需添加
