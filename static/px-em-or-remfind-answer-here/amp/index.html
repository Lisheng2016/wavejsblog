最近对px,em,rem进行应用的时候，对它们的区别有所了解但不是很深入，正好趁此机会总结一下，顺便附上[英文原文链接](https://webdesign.tutsplus.com/tutorials/comprehensive-guide-when-to-use-em-vs-rem--cms-23984?ec_unit=translation-info-language).

em,rem都是灵活的，可变化的单位，浏览器会将其转化为像素值，真实大小取决于设计时实际设置。如果使用的是1em/1rem那么浏览器可以在16-160像素中取值。

当然，在使用 pixel(px)单位时，实际的大小就是浏览器在任何情况下都会生成的效果，不会发生变化。

如何选择rem/em？

使用em,rem都可以避免提供灵活调整文字大小的能力，所以重要的是什么时候用哪个？而其中最重要的区别就是浏览器基于哪个值进行实际值的计算。

rem 计算px单位的原理：

使用rem单位，浏览器将给予网页的根元素，并根据给定的倍数计算px值，例如基于html元素。

比如根元素大小为16px，那么10rem就相当于160px,i.e. 10×16=160.


CSS padding 设定 10rem


计算值160px

em单位如何计算px值的实际大小：

当使用em单位时，计算值基于给定倍数以及第一个确定字体大小的父元素。
例如，父元素div的字体大小为18px,10em的大小为180px,i.e. 10 x 18 = 180px


CSS padding 设置为10em


计算值为180px

关键点：

关于em单位有一个比较广泛的误解：em单位计算时基于父元素的字体大小。而实际上，字体大小是相对于包含它们的元素的字体大小。

使用em单位时继承对字体的影响

使用em单位进行开发时，继承是有比较大的影响的，子元素不显式地声明自己的字体大小，就会继承父元素的字体大小。而继承的字体大小可能存在多级的比例关系，即多级的字体大小都由em单位相对大小决定，这个时候实际字体的大小就是多级相乘的结果。

假设我们的根元素字体大小为16px：
那么具体的padding大小为：
html(16px)->div(padding:1.5em)->padding实际计算值为16*1.5 = 24px;

此时在div上嵌套一个div并设置字体大小为：1.25em:
html(16px)->div(1.25em)->div(padding:1.5em)->padding实际计算值为161.251.5 = 30px;
em单位在发生嵌套时的作用可以为相乘效果。

再改变本体div的字体大小为(1.5em):
html(16px)->div(1.25em)->div(self 1.2em)->div(padding:1.5em)->padding实际计算值为161.251.2*1.5 = 36px;

当显式的设置包含文本的第一个div元素的字体大小为14px:
html(16px)->div(1.25em)->div(self 14px)->div(padding:1.5em)->padding实际计算值为14*1.5 = 21px;

浏览器的设定对HTML元素字体大小的影响

浏览器默认的字体大小往往是16px,但是可以自由更改(9px-72px),而根元素html的字体值继承自浏览器的默认设置，除非显式设置根元素的字体大小。因此,rem实际的基础值可能来自浏览器的默认设置，如果不设置HTML元素的字体大小。rem总是相对于浏览器的设定，而HTML可以继承，也可以使用em单位放大浏览器默认设置，之后 HTML子元素 使用rem单位时将相对于root element即HTML元素进行字体大小的具体计算。

当浏览设定的默认字体大小为16px,根元素大小为16px，可以方便地通过Chrome的开发者工具检查元素。
